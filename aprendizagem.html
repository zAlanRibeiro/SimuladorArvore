Pilha (Stack): O Último a Entrar é o Primeiro a Sair (LIFO)

Pense em uma pilha de pratos. Você sempre coloca um novo prato no topo e, quando precisa de um, também o retira do topo. A pilha de dados funciona da mesma forma. O último elemento que você insere é o primeiro a ser removido. Essa lógica é conhecida como LIFO (Last-In, First-Out).

Construindo uma Pilha em C

Para criar uma pilha, precisamos de duas coisas: uma estrutura para guardar cada "prato" (o dado em si) e outra para gerenciar a pilha como um todo.

A Estrutura do Elemento (o "Prato")

Cada elemento da nossa pilha precisa guardar um valor (um número, por exemplo) e uma "seta" que aponte para o elemento que está abaixo dele na pilha. Em C, criamos isso com uma struct.

// Cada elemento (ou nó) da pilha
struct Elemento {
    int num; // O dado que vamos guardar
    struct Elemento *prox; // Ponteiro para o próximo elemento abaixo
};
A Estrutura da Pilha (a "Mesa" que segura os pratos)

Agora, precisamos de uma estrutura que controle a pilha inteira. Ela só precisa de uma coisa: saber quem está no topo. Quando a pilha estiver vazia, o topo apontará para NULL (nulo).

// A estrutura que gerencia a pilha
struct Pilha {
    struct Elemento *topo; // Ponteiro que sempre aponta para o topo da pilha
};

Operações Principais da Pilha

Empilhar (push)

Para adicionar um novo elemento, seguimos estes passos:

  • Criamos um novo Elemento.
  • Fazemos o ponteiro prox desse novo elemento apontar para o antigo topo.
  • Atualizamos o topo da pilha para ser este novo elemento.

Veja como fica a função em C:

// Função para inserir um elemento no topo da pilha (push)
void insere_na_pilha(struct Pilha *pi, int numero) {
    struct Elemento *novo = (struct Elemento*) malloc(sizeof(struct Elemento));
    if (novo != NULL) {
        novo->num = numero;
        novo->prox = pi->topo;
        pi->topo = novo;
        printf("Número %d inserido na pilha!\n", numero);
    }
}
Desempilhar (pop)

Para remover o elemento do topo, fazemos o seguinte:

  • Verificamos se a pilha não está vazia.
  • Guardamos o endereço do topo atual em um ponteiro auxiliar.
  • Fazemos o topo da pilha apontar para o elemento que estava abaixo dele.
  • Liberamos a memória do elemento que foi removido (o antigo topo).

A função em C fica assim:

// Função para remover um elemento do topo da pilha (pop)
void remove_da_pilha(struct Pilha *pi) {
    if (pi->topo == NULL) {
        printf("Pilha Vazia!!\n");
    } else {
        struct Elemento *aux = pi->topo;
        printf("Número %d removido!\n", pi->topo->num);
        pi->topo = pi->topo->prox;
        free(aux);
    }
}

Fila (Queue): O Primeiro a Entrar é o Primeiro a Sair (FIFO)

Pense em uma fila de supermercado. A primeira pessoa que chega ao caixa é a primeira a ser atendida e a ir embora. A estrutura de dados de fila funciona exatamente assim. O primeiro elemento que você insere é o primeiro que será removido. Essa lógica é conhecida como FIFO (First-In, First-Out).

Construindo uma Fila em C

Para criar uma fila, a estrutura do elemento é a mesma da pilha. O que muda é a estrutura de gerenciamento, que agora precisa controlar duas pontas: o início e o fim da fila.

A Estrutura do Elemento (a "Pessoa" na fila)

Assim como na pilha, cada elemento da fila guarda um valor e um ponteiro que aponta para o próximo elemento. Usamos a mesma struct.

// Cada elemento (ou nó) da fila
    struct Elemento {
        int num; // O dado que vamos guardar
        struct Elemento *prox; // Ponteiro para o próximo elemento na fila
    };
A Estrutura da Fila (o controle do "Caixa")

Para gerenciar a fila, precisamos de dois ponteiros: um para o inicio (o próximo a ser atendido) e um para o fim (onde o próximo elemento vai entrar). Quando a fila está vazia, ambos apontam para NULL.

// A estrutura que gerencia a fila
    struct Fila {
        struct Elemento *inicio; // Ponteiro para o início da fila
        struct Elemento *fim;    // Ponteiro para o fim da fila
    };

Operações Principais da Fila

Enfileirar (enqueue)

Para adicionar um novo elemento ao final da fila, seguimos estes passos:

  • Criamos um novo Elemento.
  • Se a fila estiver vazia, o inicio e o fim apontarão para este novo elemento.
  • Se não, o ponteiro prox do antigo fim apontará para o novo elemento.
  • Por último, atualizamos o fim da fila para ser este novo elemento.

Veja como fica a função em C:

// Função para inserir um elemento no fim da fila (enqueue)
    void insere_na_fila(struct Fila *fi, int numero) {
        struct Elemento *novo = (struct Elemento*) malloc(sizeof(struct Elemento));
        if (novo != NULL) {
            novo->num = numero;
            novo->prox = NULL; // O último da fila sempre aponta para NULL

            if (fi->fim == NULL) { // Se a fila está vazia
                fi->inicio = novo;
            } else {
                fi->fim->prox = novo;
            }
            fi->fim = novo;
            printf("Número %d inserido na fila!\n", numero);
        }
    }
Desenfileirar (dequeue)

Para remover o elemento do início da fila, fazemos o seguinte:

  • Verificamos se a fila não está vazia.
  • Guardamos o endereço do inicio atual em um ponteiro auxiliar.
  • Fazemos o inicio da fila apontar para o elemento seguinte (inicio->prox).
  • Se, após a remoção, a fila ficar vazia, também atualizamos o fim para NULL.
  • Liberamos a memória do elemento que foi removido (o antigo início).

A função em C fica assim:

// Função para remover um elemento do início da fila (dequeue)
    void remove_da_fila(struct Fila *fi) {
        if (fi->inicio == NULL) {
            printf("Fila Vazia!!\n");
        } else {
            struct Elemento *aux = fi->inicio;
            printf("Número %d removido!\n", fi->inicio->num);
            fi->inicio = fi->inicio->prox;

            if (fi->inicio == NULL) { // Se a fila ficou vazia
                fi->fim = NULL;
            }
            free(aux);
        }
    }

Entendendo as Árvores Binárias

Em computação, uma "árvore" é uma estrutura de dados que simula uma hierarquia, como uma árvore genealógica. Cada elemento, chamado de nó, pode ter "filhos", que são outros nós ligados a ele.

O que é uma Árvore Binária?

Uma árvore binária é um tipo especial de árvore onde cada nó pode ter, no máximo, dois filhos: um à esquerda e um à direita. O nó no topo da hierarquia, que não tem "pai", é chamado de raiz (root).

Diagrama mostrando a estrutura de uma árvore binária simples, destacando a raiz, nós internos, nós folha, e os filhos da esquerda e da direita.

Árvores Binárias de Busca (BST - Binary Search Tree)

Uma variação muito importante é a Árvore Binária de Busca. Ela impõe uma regra de ordem para os seus elementos:

  • Todos os valores na sub-árvore esquerda de um nó são menores que o valor do nó.
  • Todos os valores na sub-árvore direita de um nó são maiores que o valor do nó.

Essa regra torna a busca por elementos extremamente eficiente, similar a uma pesquisa binária.

O Problema: Árvores Desbalanceadas

A eficiência de uma BST depende de sua altura. Se inserirmos elementos de forma ordenada (ex: 10, 20, 30, 40), a árvore se degenera e vira, na prática, uma lista ligada. A busca, que deveria ser rápida, passa a ter o mesmo desempenho de uma busca linear.

Comparação lado a lado: uma árvore de busca balanceada (com aparência de "árvore") e uma desbalanceada (com aparência de "lista").

A Solução: Árvores AVL

Uma Árvore AVL é uma Árvore Binária de Busca autobalanceável. Ela garante que a árvore permaneça sempre o mais "cheia" e "compacta" possível, evitando que se degenere em uma lista. O nome AVL vem de seus inventores: Adelson-Velsky e Landis.

O Fator de Balanceamento

A mágica da AVL está no controle do Fator de Balanceamento (FB) de cada nó. Ele é calculado da seguinte forma:

FB = (Altura da sub-árvore da esquerda) - (Altura da sub-árvore da direita)

Em uma árvore AVL, o Fator de Balanceamento de todos os nós deve ser sempre -1, 0 ou 1. Se, após uma inserção ou remoção, o FB de algum nó se torna -2 ou 2, a árvore está desbalanceada e precisa de um reajuste.

O Reajuste: Rotações

Quando a árvore fica desbalanceada, a AVL executa operações chamadas rotações para corrigir a estrutura. Existem quatro tipos de rotações, que resolvem todos os casos de desbalanceamento.

1. Rotação Simples à Direita (Caso Esquerda-Esquerda)

Ocorre quando um novo nó é inserido na sub-árvore esquerda do filho esquerdo de um nó desbalanceado.

Animação ou diagrama "antes e depois" mostrando uma Rotação Simples à Direita.

2. Rotação Simples à Esquerda (Caso Direita-Direita)

Ocorre quando um novo nó é inserido na sub-árvore direita do filho direito de um nó desbalanceado.

Animação ou diagrama "antes e depois" mostrando uma Rotação Simples à Esquerda.

3. Rotação Dupla à Direita (Caso Esquerda-Direita)

Ocorre quando um novo nó é inserido na sub-árvore direita do filho esquerdo. É uma Rotação à Esquerda seguida de uma Rotação à Direita.

Animação ou diagrama "antes e depois" mostrando uma Rotação Dupla à Direita.

4. Rotação Dupla à Esquerda (Caso Direita-Esquerda)

Ocorre quando um novo nó é inserido na sub-árvore esquerda do filho direito. É uma Rotação à Direita seguida de uma Rotação à Esquerda.

Animação ou diagrama "antes e depois" mostrando uma Rotação Dupla à Esquerda.

Implementação em C

Para implementar uma árvore AVL, a estrutura do nó precisa armazenar, além dos ponteiros para os filhos, a sua altura (para facilitar o cálculo do Fator de Balanceamento).

// Estrutura do nó de uma árvore AVL
typedef struct no_avl {
    int valor;
    int altura; // Altura da sub-árvore com raiz neste nó
    struct no_avl *esquerda;
    struct no_avl *direita;
} NoAVL;

As funções de inserção e remoção verificam o Fator de Balanceamento após cada operação e disparam as rotações necessárias para manter a propriedade da AVL.

Desvendando as Listas Ligadas (Linked Lists)

Diferente de um array, que armazena dados em um bloco contínuo de memória, uma Lista Ligada (ou Encadeada) é uma coleção de elementos, chamados de nós, que podem estar espalhados pela memória. A mágica está em como eles se conectam: cada nó guarda, além do seu próprio dado, um "endereço" que aponta para o próximo nó da sequência.

O que é um Nó?

O "nó" (ou `node` em inglês) é o bloco de construção fundamental de uma lista. Ele é composto por duas partes principais:

  • Dado: A informação que queremos armazenar (um número, um nome, etc.).
  • Ponteiro (ou referência): Um endereço de memória que "aponta" para o próximo nó da lista. O ponteiro do último nó aponta para NULL, indicando o fim da sequência.

Em C, representamos um nó com uma struct:

// Estrutura de um nó da lista
typedef struct no {
    int dado;               // Onde o valor é armazenado
    struct no *proximo;    // Ponteiro para o próximo nó
} No;

lista com apenas um nó apontando para NULL.

A Lista Completa

A lista inteira é gerenciada através de um único ponteiro, geralmente chamado de inicio ou head, que aponta apenas para o primeiro nó. A partir dele, conseguimos percorrer toda a lista, seguindo os ponteiros de um nó para o outro até encontrar um NULL.

Lista completa sendo o ponteiro do ultimo nó apontando para NULL.

Operações Fundamentais

Inserção de um Novo Nó

Inserir um elemento é uma das grandes vantagens da lista ligada, pois não precisamos deslocar outros elementos como em um array. Apenas manipulamos os ponteiros.

Para inserir no início (a operação mais eficiente):

  1. Cria-se um novo nó.
  2. O ponteiro "próximo" do novo nó aponta para o antigo primeiro nó (o atual inicio).
  3. O ponteiro inicio da lista é atualizado para apontar para este novo nó.
// Função para inserir um nó no início da lista
void inserir_no_inicio(No **inicio, int valor) {
    No *novo_no = malloc(sizeof(No));
    if (novo_no) {
        novo_no->dado = valor;
        novo_no->proximo = *inicio;
        *inicio = novo_no;
    }
}

Passo a passo de uma inserção em uma lista simples.

Remoção de um Nó

Remover um nó também envolve a manipulação de ponteiros. Para remover o primeiro nó, por exemplo:

  1. Guarda-se o endereço do segundo nó em um ponteiro auxiliar.
  2. Libera-se a memória do primeiro nó (o atual inicio).
  3. O ponteiro inicio da lista é atualizado para apontar para o segundo nó.

Tipos de Listas Ligadas

Além da lista que vimos (Simplesmente Ligada), existem outras variações importantes:

  • Lista Duplamente Ligada: Cada nó tem dois ponteiros: um para o próximo nó e outro para o nó anterior. Isso permite percorrer a lista em ambas as direções (frente e trás), mas consome um pouco mais de memória.
  • Lista Circular: O ponteiro do último nó, em vez de apontar para NULL, aponta de volta para o primeiro nó, criando um ciclo.

lista duplamente ligada e uma circular.

Vantagens vs. Desvantagens

Vantagens:
  • Tamanho Dinâmico: Crescem e diminuem facilmente em tempo de execução.
  • Inserção/Remoção Eficiente: Especialmente no início e fim, essas operações são muito rápidas (O(1)).
Desvantagens:
  • Acesso Sequencial: Para acessar um elemento no meio da lista, é preciso percorrer todos os nós desde o início (O(n)), diferente de um array que tem acesso direto (O(1)).
  • Mais Memória: Gastam memória extra para armazenar os ponteiros.
